# official-main-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，不依赖后端服务，可直接通过浏览器访问。

## 目录说明

```
.
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，命名清晰，便于查找。
- **assets/**：存放页面间共享的样式、脚本或媒体文件，避免重复引用。
- **index.html**：可选的入口页面，用于展示归档索引或导航。

## 页面归档说明

- 每个 HTML 页面均为独立完整文档，包含自身所需的全部结构与样式。
- 页面之间无依赖关系，可单独打开运行。
- 归档内容可能包含信息展示、工具界面、演示页面等，不限定用途。

## 维护说明

- 新增页面请放入 `pages/` 目录，并保持命名规范（如 `page-name.html`）。
- 如需引用共享资源，请使用相对路径指向 `assets/`。
- 不建议在页面内嵌入外部不可控资源，以保证离线或镜像环境下的可用性。
- 本仓库不跟踪具体域名或网站内容，仅作为静态 HTML 的托管与存档点。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器打开 `pages/` 下的任意 HTML 文件。
3. 或通过 GitHub Pages 等服务部署后访问（需自行配置）。
